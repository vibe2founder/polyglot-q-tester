# Relatório de Implementação - assertLatency

## O que foi feito

- Implementado o método `assertLatency` na classe `TestScenario`.
- Atualizado o método `actEmit` para retornar o timestamp da emissão do evento.
- Atualizado o método `assertReceived` para retornar o objeto da mensagem encontrada, facilitando asserções subsequentes.
- Corrigidas as importações relativas (`../core/index`) nos arquivos `scenario.ts` e `mock-adapter.ts`, que agora apontam corretamente para `../one-q-4-all/packages/core/index` (baseado na estrutura do monorepo).
- Criado arquivo de teste `tests/latency.test.ts` para validar o comportamento do `assertLatency` (sucesso e falha).

## Por que foi feito

- A funcionalidade de `assertLatency` foi solicitada para permitir a verificação de requisitos não-funcionais (performance) nos testes de integração do sistema de filas.
- A correção dos imports foi necessária para execução dos testes, já que a referência original estava quebrada.

## Como testar

Executar o comando:
```bash
bun test tests/latency.test.ts
```
